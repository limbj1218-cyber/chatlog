# 🚀 단톡방 일일 정리 페이지 — 시작 가이드

[meoksal 페이지](https://flich0600-source.github.io/meoksal/)와 같은 방식으로 동작하는 프로젝트입니다.

## 원리 (분석 결과)

원본 페이지는 생각보다 단순합니다:

1. **서버도, 데이터베이스도, 자동 연동도 없습니다.** `index.html` 파일 하나에 모든 내용이 하드코딩된 순수 정적 페이지입니다.
2. 운영자가 매일 카카오톡 대화를 내보내서 → AI(Claude 등)로 요약 → 요약 결과를 HTML에 추가 → GitHub에 push 하는 방식입니다. (커밋 이력이 매일 1~6회, "7/9 새벽반 정리 추가 + 7/8 전체 재분석" 같은 메시지로 확인됨)
3. **GitHub Pages**가 이 HTML을 무료로 호스팅합니다. push하면 웹페이지가 자동 갱신됩니다.
4. 크롬에 띄워둔 페이지는 새로고침하면 최신 내용이 보입니다.

즉 "자동"처럼 보이지만 실제로는 **매일 5분짜리 수동 루틴 + AI 요약**입니다. 카카오톡은 대화를 읽어가는 공식 API가 없어서 완전 자동화는 불가능하고, 이 방식이 현실적인 최선입니다.

## 1회만 하면 되는 초기 설정

### ① GitHub 저장소 만들기
1. https://github.com 가입/로그인
2. 새 저장소 생성 (예: `chatlog`), Public으로 설정
3. 이 폴더(D:\Chatbot)에서:
   ```
   git init
   git add -A
   git commit -m "최초 배포"
   git branch -M main
   git remote add origin https://github.com/내아이디/chatlog.git
   git push -u origin main
   ```

### ② GitHub Pages 켜기
1. 저장소 → Settings → Pages
2. Source: `Deploy from a branch`, Branch: `main` / `(root)` 선택 → Save
3. 1~2분 후 `https://내아이디.github.io/chatlog/` 에서 페이지 확인

### ③ 자동화 켜기 (1회만)

Windows 작업 스케줄러에 감시 스크립트를 등록합니다. PowerShell을 열고:

```
schtasks /create /tn "카톡정리" /tr "powershell -ExecutionPolicy Bypass -WindowStyle Hidden -File D:\Chatbot\자동정리.ps1" /sc minute /mo 15 /f
```

이러면 15분마다 `자동정리.ps1`이 실행되어 `chats\` 폴더를 확인하고, **새 대화 파일이 있을 때만** Claude로 정리 → index.html 갱신 → push 까지 자동으로 합니다. 새 파일이 없으면 아무것도 하지 않으므로 비용도 들지 않습니다.

- 해제하려면: `schtasks /delete /tn "카톡정리" /f`
- 실행 기록은 `자동정리.log` 파일에서 확인

## 매일 하는 일 (30초, 이것만 수동)

**PC 카카오톡**: 채팅방 → 우측 상단 ≡ 메뉴 → **대화 내용 → 대화 내보내기** → 저장 위치를 `D:\Chatbot\chats\`로 지정. 파일명은 카톡이 만들어주는 기본 이름 그대로 둬도 됩니다.

이게 전부입니다. 15분 안에 스케줄러가 새 파일을 감지해서 정리·배포하고, 크롬에 띄워둔 페이지는 10분마다 스스로 새로고침되므로 자동으로 최신 내용이 보입니다.

> **왜 내보내기는 자동이 안 되나요?** 카카오톡은 대화를 읽어가는 공식 API를 제공하지 않습니다. 화면 클릭을 흉내내는 매크로(AutoHotkey 등)로 내보내기까지 자동화할 수는 있지만 카톡 업데이트 때마다 깨지기 쉽고, 안드로이드 폰의 알림을 읽어 실시간 수집하는 방법(메신저봇R 등)은 상시 켜진 폰이 필요하고 카카오 약관상 회색지대라 권하지 않습니다.

## 페이지 구성 (원본 사이트와 동일한 구조)

- **고정 공지**: 항상 위에 보이는 공지 배너
- **날짜별 카드** (최신이 맨 위, 클릭하면 접기/펼치기):
  - 통계 (메시지 수 / Q&A 수 / 인사이트 수)
  - 📢 주요 공지
  - 🏆 오늘의 MVP
  - ❓ 주요 Q&A
  - 💡 인사이트 & 팁
  - 🕒 타임라인

## 커스터마이징

- 제목/부제: `index.html` 의 `<header>` 부분 수정
- 색상: `<style>` 맨 위 `:root` 변수 수정 (`--accent`가 포인트 색)
- 섹션 추가/삭제: Claude에게 "용어사전 섹션도 추가해줘" 같이 말하면 됨
