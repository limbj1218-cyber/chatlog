# 🤖 단톡봇 설치 가이드 (메신저봇R)

카카오톡 **오픈채팅방**에서 `/명령어`에 반응하고, `/벽타기`로 대화 정리 페이지를 게시하는 봇입니다.
봇 스크립트는 [단톡봇.js](단톡봇.js) 에 완성되어 있습니다.

## 동작 원리

카카오톡은 단톡방용 공식 봇 API가 없습니다. 메신저봇R은 이렇게 우회합니다:

1. 안드로이드 폰이 카카오톡 **알림**을 읽는다 (알림 접근 권한)
2. 메시지 내용을 스크립트(`단톡봇.js`)에 전달한다
3. 스크립트가 답장을 만들면 알림의 **빠른 답장** 기능으로 전송한다

즉 **봇 = 카톡이 로그인된 안드로이드 폰 그 자체**입니다. 그래서:

- 폰이 항상 켜져 있고 인터넷에 연결되어 있어야 함 (충전기 꽂아두기)
- 봇이 답할 방의 **알림이 켜져 있어야** 함
- 답장은 그 폰에 로그인된 계정 이름으로 나감 → **남는 폰 + 별도 계정** 권장
- 카카오 약관상 회색지대입니다. 소규모 개인 용도로만 쓰세요.

## 권한 체계

| 역할 | 누구 | 할 수 있는 것 |
|---|---|---|
| **최고 관리자** | 대화명에 `후파` 포함 | 전부 (`/지정` `/해제` 포함) |
| **부관리자** | 방에서 `/지정 이름조각` 으로 지정 | 그 방의 `/등록` `/삭제` |
| 일반 | — | 자동응답 사용, `/벽타기` `/목록` `/통계` `/날씨` `/봇` |

- `/지정 뿡재` → 대화명에 "뿡재"가 포함된 사람(예: `이@재(뿡재)`)이 **그 방의** 부관리자가 됩니다.
- ⚠️ 이름 "포함" 방식이라 누군가 닉네임에 `후파`를 넣으면 관리자 사칭이 가능합니다. 문제가 생기면 정확히 일치 방식으로 바꿔달라고 요청하세요.

## 명령어 정리

### 방 등록 — 스크립트 맨 위 ROOMS 목록에 직접 입력

```js
var ROOMS = [
    "임병진",
    "우리 오픈채팅방",   // ← 카톡에 보이는 방 이름과 정확히 일치, 쉼표로 계속 추가
];
```

목록에 없는 방은 봇이 완전히 무시합니다. 수정 후 재컴파일 필요.

### 활성화된 방에서 (모든 데이터는 방별로 분리)

| 명령어 | 권한 | 동작 |
|---|---|---|
| `/등록 안녕 하세요 좋은날씨 입니다.` | 관리자 | 자동응답 등록. 첫 단어(`안녕`)가 트리거, **나머지 전부**(띄어쓰기 포함)가 내용 |
| (누군가 `안녕` 이라고 정확히 입력) | 누구나 | 봇이 `하세요 좋은날씨 입니다.` 라고 응답 |
| `/삭제 안녕` | 관리자 | 자동응답 삭제 |
| `/공지 내용...` | 관리자 | 페이지 상단 고정 공지 등록 (다음 정리 때 반영) |
| `/목록` | 누구나 | 이 방의 자동응답 목록 |
| `/벽타기` | 누구나 | 오늘 대화를 정리해서 이 방 전용 페이지에 게시 (**6시간에 1번**) |
| `/지정 이름` / `/해제 이름` / `/관리자` | 최고 관리자 | 부관리자 관리 |
| `/통계` | 누구나 | 오늘 메시지 수·참여 인원·수다왕 TOP3 |
| `/날씨 지역` | 누구나 | 현재 날씨 |
| `/봇` `/도움말` | 누구나 | 상태 확인 / 명령어 안내 |
| `/방이름` | 누구나 (모든 방) | 진단: 봇이 보는 방 이름·활성화 여부·파일 저장 상태 |

> **저장 방식**: 자동응답·부관리자·대화기록은 파일로 저장을 시도하고, 폰 저장소 권한이 없으면
> 자동으로 메모리 모드로 동작합니다. 메모리 모드에서는 메신저봇R을 재시작하면 자동응답/부관리자/
> 오늘 기록이 사라지므로, 가능하면 폰 설정에서 메신저봇R에 "모든 파일 접근" 권한을 주세요.

자동응답은 **메시지 전체가 트리거와 정확히 일치**할 때만 반응합니다 ("안녕하세요~"에는 반응 안 함).

## 🧗 벽타기 구조 (PC 꺼져 있어도 동작)

```
로그 업로드 (아래 3가지 경로 중 하나)
  → GitHub Actions가 클라우드에서 Claude 실행, 그 방 페이지(rooms/방이름/) 갱신
  → GitHub Pages 자동 반영 (약 2~5분)
```

페이지가 갱신되는 3가지 경로 (전부 같은 페이지):

| 경로 | 시점 | 조건 |
|---|---|---|
| 수동 `/벽타기` | 누가 명령 입력 시 | 방별 6시간 쿨다운 |
| 자동 벽타기 | 02:50 / 08:50 / 14:50 / 20:50 | 새 대화가 있을 때만 |
| 전날 마감 | 자정 넘어 첫 메시지 도착 시 | 전날 로그를 최종본으로 업로드 |

그리고 **09:00 / 15:00 / 21:00**에는 봇이 방에 정리 페이지 링크를 공지합니다 (지난 공지 이후 새 대화가 있었을 때만 — 도배 방지). 시간을 바꾸려면 스크립트 상단 `AUTO_UPLOAD_TIMES` / `ANNOUNCE_TIMES` 를 수정하세요.

> 링크 공지는 봇 폰이 그 방의 카톡 알림을 최소 한 번 받은 뒤부터 보낼 수 있습니다 (안드로이드 구조상 제약). 폰을 재부팅한 직후 방이 조용하면 첫 공지가 건너뛰어질 수 있어요.

## 설치

### 1단계 — GitHub 준비 (PC에서, 1회)

1. https://github.com 에서 Public 저장소 생성 (예: `chatlog`)
2. 이 폴더(D:\Chatbot)를 push:
   ```
   git init
   git add -A
   git commit -m "최초 배포"
   git branch -M main
   git remote add origin https://github.com/내아이디/chatlog.git
   git push -u origin main
   ```
3. 저장소 → Settings → Pages → Source: `Deploy from a branch`, `main` / `(root)` → Save
4. **Claude 인증 등록** (Actions가 Claude를 돌릴 때 사용, 둘 중 하나):
   - **A. 구독 사용 (Pro/Max, 추가 비용 없음)**: PC 터미널에서 `claude setup-token` 실행 → 나온 토큰을 저장소 Settings → Secrets and variables → Actions → New secret, 이름 `CLAUDE_CODE_OAUTH_TOKEN` 으로 등록. 그리고 `.github/workflows/summarize.yml` 에서 주석 안내대로 한 줄 교체.
   - **B. API 키 (사용량 과금)**: https://console.anthropic.com 에서 키 발급 → 시크릿 이름 `ANTHROPIC_API_KEY` 로 등록.
5. **봇용 토큰(PAT) 발급**: GitHub → Settings(개인) → Developer settings → Fine-grained tokens → Generate:
   - Repository access: 이 저장소만
   - Permissions: **Contents → Read and write** (그 외 전부 No access)
   - 발급된 토큰을 복사해 둠 (폰 스크립트에 넣을 것)

### 2단계 — 폰 설정 (1회)

1. 봇 폰에 Google Play에서 "메신저봇R" 설치 (없으면 개발자 공식 채널의 APK만 사용)
2. 앱 안내에 따라 **알림 접근 권한**, **저장소 권한** 허용
3. 카카오톡 로그인 (봇 계정) → 대상 방들의 **알림 켜기**
4. 설정 → 배터리 → 메신저봇R·카카오톡 **배터리 최적화 제외**, 방해금지 해제

### 3단계 — 스크립트 등록 (로더 방식)

폰에는 작은 **로더**([로더.js](로더.js))만 넣습니다. 로더가 GitHub에서 본체 코드(단톡봇.js)를
읽어와 실행하므로, 본체가 갱신되면 방에서 `/업데이트` 한 번이면 폰에 반영됩니다.
**토큰과 방 목록은 로더(폰)에만 존재**해서 공개 저장소에 노출되지 않습니다.

1. 메신저봇R → ➕ 새 봇 (이름: `단톡봇`, JavaScript)
2. `로더.js` 내용 전체를 붙여넣기 (폰 브라우저에서 GitHub의 bot/로더.js → Raw → 전체 복사)
3. 맨 위 두 값만 수정:
   ```js
   var TOKEN = "github_pat_...";   // 1단계 5번에서 발급한 토큰
   var MY_ROOMS = [
       "우리 오픈채팅방",           // 봇이 동작할 방 (카톡 이름과 정확히 일치)
   ];
   ```
4. **컴파일** → 성공 확인 → 전원 토글 ON

> 본체(단톡봇.js)를 직접 붙여넣는 방식도 여전히 가능하지만, 그 경우 코드 갱신 때마다
> 전체를 다시 붙여넣고 TOKEN/ROOMS를 재입력해야 합니다.

**로더 전용 명령어**: `/업데이트` (깃헙 최신 코드 다시 불러오기, 최고 관리자만), `/로더` (로더 상태 확인)

### 4단계 — 테스트

1. "임병진" 채팅방에서 `/등록 테스트방이름` → "✅ 봇 활성화" 응답 확인
2. 그 방에서 (다른 사람이) `/봇` → "🤖 봇 정상 동작 중" 확인
3. `/등록 안녕 반갑습니다` → `안녕` 입력 → "반갑습니다" 응답 확인
4. 대화 몇 개 쌓은 뒤 `/벽타기` → 2~5분 후 링크 접속해 페이지 확인

## 문제 해결

- **봇이 반응 없음**: ① 전원 토글 ON ② 컴파일 성공 ③ 방 알림 켜짐 ④ 배터리 최적화 제외 순서로 확인
- **봇 폰 계정이 보낸 메시지에 반응 안 함**: 정상입니다. 알림 기반이라 자기 메시지는 못 읽습니다. 다른 사람이 보내야 반응합니다.
- **제어실에서 등록했는데 그 방에서 조용함**: 방 이름이 한 글자라도 다르면 안 됩니다. `/방목록`으로 등록된 이름과 카톡 방 이름을 대조하세요.
- **벽타기 업로드 실패**: 폰 인터넷, GITHUB.TOKEN 만료 여부, OWNER/REPO 오타 확인
- **벽타기는 됐는데 페이지가 안 바뀜**: GitHub 저장소 → Actions 탭에서 `wall-climb` 실행 로그 확인 (Claude 시크릿 미등록이 흔한 원인)
- **날씨 안 나옴**: wttr.in 일시 장애일 수 있음. 잠시 후 재시도.
